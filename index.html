  • 🪣 Deploying a Static Website on AWS S3 Using Terraform for Infrastructure as Code

    This project showcases how to automate the setup of a static website using Terraform and Amazon S3. The Terraform configuration handles the creation of an S3 bucket, sets it up for static website hosting, and configures public access so users can view the website directly via the S3 endpoint. With a few simple Terraform commands, the infrastructure is deployed automatically, allowing you to focus on your content instead of manual setup. After deployment, you can upload your HTML, CSS, and JavaScript files to the S3 bucket, and your static website is live.

    Key Features:

    • Infrastructure as Code with Terraform
    • Static website hosting on AWS S3
    • Public access configuration
    • Easy deployment and cleanup

      • terraform plan

      • terraform apply
    • Upload your website files to the S3 bucket manually after creation.
    • Clean up when done:
      • terraform destroy

    • Create a new Jenkins pipeline job and link it to your GitHub repository.​

    • Launch an EC2 instance with Docker installed.​
    • Ensure security groups allow SSH (port 22) and HTTP (port 80) access.​

    • Trigger the Jenkins pipeline manually or set up webhook triggers for automatic deployment on code push.​

  • 🧩 Microservice Deployment on Kubernetes Using Minikube, kubectl, YAML, and Helm

    Project Description:

    This project demonstrates the deployment of a simple microservice within a local Kubernetes cluster using Minikube. The deployment process utilizes kubectl commands, YAML manifests, and Helm charts to manage Kubernetes resources effectively.​

    Key Components:

    • Minikube: Provides a local Kubernetes environment for testing and development purposes.​
    • kubectl: The command-line tool used to interact with the Kubernetes cluster, facilitating the management of cluster resources.​
    • YAML Manifests: Define the desired state of Kubernetes resources such as Deployments and Services.​
    • Helm: A package manager for Kubernetes that simplifies the deployment and management of applications through reusable charts.​

    Deployment Workflow:

    1. Set Up Minikube:
      • Install Minikube on your local machine.
      • Start the Minikube cluster:
        • minikube start
    2. Deploy Microservice Using YAML:
      • Create a deployment.yaml file defining the Deployment resource.
      • Create a service.yaml file defining the Service resource.
      • Apply the configurations:
        • 
        kubectl apply -f deployment.yaml
        kubectl apply -f service.yaml
    3. Access the Microservice:
      • Retrieve the URL to access the service:
        • minikube service your-service-name --url
    4. Deploy Using Helm:
      • Create a Helm chart for the microservice.
      • Install the Helm chart:
        • helm install your-release-name ./your-chart-directory

    Skills Demonstrated:

    • Proficiency in setting up and managing a local Kubernetes cluster using Minikube.​
    • Ability to define and apply Kubernetes resources using YAML manifests.​
    • Experience with Helm for packaging and deploying applications on Kubernetes.​
    • Familiarity with kubectl for cluster interaction and resource management.​
